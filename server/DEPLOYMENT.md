# 🚀 Инструкция по развёртыванию на Railway

## 1️⃣ Подготовка

### Файлы для деплоя:
```
server/
├── server.js          # Основной файл сервера
├── package.json       # Зависимости Node.js
├── .gitignore        # Игнорируемые файлы
└── messenger.db      # База данных (создаётся автоматически)
```

## 2️⃣ Развёртывание на Railway

### Вариант A: Через GitHub
1. Создайте репозиторий на GitHub
2. Запушьте код (кроме `messenger.db`)
3. Подключите репозиторий к Railway
4. Railway автоматически определит `package.json` и запустит сервер

### Вариант B: Через Railway CLI
```bash
# Установите CLI
npm i -g @railway/cli

# Войдите в аккаунт
railway login

# Инициализируйте проект
railway init

# Задеплойте
railway up
```

## 3️⃣ Настройка переменных окружения

В панели Railway добавьте переменные:
```bash
NODE_ENV=production
PORT=3000  # Railway автоматически назначит порт
```

## 4️⃣ Важно!

### ✅ База данных сохраняется
- Все пользователи сохраняются в `messenger.db`
- При перезапуске сервера данные не теряются
- База данных автоматически создаётся при первом запуске

### ⚠️ Файлы которые НЕ нужно загружать:
- `messenger.db` (создаётся автоматически)
- `node_modules/` (устанавливается автоматически)
- `.env` (используйте переменные окружения Railway)

### 🔒 Безопасность
- В production включите WSS (HTTPS)
- Настройте `ALLOWED_ORIGINS` в `server.js`
- Используйте переменные окружения для чувствительных данных

## 5️⃣ Проверка работы

После деплоя:
1. Откройте веб-приложение
2. Зарегистрируйте нового пользователя
3. Перезапустите сервер в панели Railway
4. Войдите под тем же пользователем - **должно работать!**

## 6️⃣ Мониторинг

### Логи в Railway:
```bash
railway logs
```

### Проверка базы данных:
```bash
# Подключитесь к серверу
railway run bash

# Проверьте базу
sqlite3 messenger.db "SELECT * FROM users;"
```

## 7️⃣ Обновление

```bash
# Запушьте изменения в GitHub
git push

# Railway автоматически обновит сервер
# Или вручную:
railway up
```

## 📊 Структура базы данных

### Таблица `users`:
- `username` - имя пользователя (PRIMARY KEY)
- `passwordHash` - хеш пароля
- `salt` - соль для хеширования
- `createdAt` - дата создания
- `lastLogin` - последний вход
- `isVisibleInDirectory` - видимость в списке
- `allowGroupInvite` - разрешение на приглашение в группы

### Таблица `groups`:
- `id` - ID группы
- `name` - название
- `creator` - создатель
- `createdAt` - дата создания
- `lastMessage` - последнее сообщение

### Таблица `group_members`:
- `groupId` - ID группы
- `username` - имя участника
